# ☁️ 클라우드 컴퓨팅이란? 입문자를 위한 개념 정리

안녕하세요! 이번 글에서는 **클라우드 컴퓨팅(Cloud Computing)** 의 개념부터 활용 모델까지 정리해보겠습니다. 최근 AWS가 Amazon 전체 이익의 60% 이상을 차지했다는 사실, 알고 계셨나요? 클라우드는 단순한 유행이 아니라 이미 기업의 핵심 인프라로 자리 잡고 있습니다. 그럼 왜 클라우드가 그렇게 중요한 기술일까요?

## 1. 클라우드 컴퓨팅이란?

### ✅ 정의
> 클라우드 컴퓨팅은 인터넷을 통해 컴퓨팅 자원(서버, 저장소, 데이터베이스, 네트워킹 등)을 필요한 만큼 제공받고, 사용한 만큼만 비용을 지불하는 방식입니다.

더 이상 비싼 서버를 사서 사무실에 설치할 필요 없이, Amazon Web Services(AWS), Microsoft Azure, Google Cloud Platform(GCP) 같은 클라우드 제공 업체에서 렌탈처럼 자원을 사용할 수 있죠.

## 2. 클라우드가 필요한 이유
### 💻 예시: DataCamp 서버 문제

DataCamp처럼 교육 플랫폼을 운영한다고 가정해 봅시다. "무료 체험 주간"에 접속자가 폭증하면 서버 과부하로 사이트가 느려지거나 중단될 수 있어요. 기존 방식대로라면 서버를 추가 구매해야 하지만, 이건 너무 비쌉니다.

### ☁️ 클라우드의 등장
> 클라우드는 트래픽이 급증하면 자동으로 서버 자원을 늘리고, 다시 줄어들면 비용도 줄어드는 유연성을 제공합니다. 즉, 클라우드는 전기처럼 “쓴 만큼만 요금 청구” 되는 시스템입니다.

# 🌐 클라우드는 어떻게 구현될까? — 클라우드 배포 모델과 규제, 직무 이해

앞서 클라우드 컴퓨팅의 기본 개념과 장점을 이해했다면, 이제는 실제 기업들이 클라우드를 어떻게 도입하는지, 그리고 도입 시 어떤 제약과 역할들이 필요한지에 대해 알아볼 차례입니다.

## 1️⃣ 클라우드 배포 모델 (Cloud Deployment Models)
💡 왜 중요할까?
클라우드를 도입할 때 가장 먼저 결정해야 하는 것은 어떤 방식으로 클라우드를 사용할지, 즉 배포 모델입니다. 이는 조직의 보안 요구사항, 예산, 데이터 통제 수준에 따라 달라집니다.

### 🔐 Private Cloud (프라이빗 클라우드)
- 특징: 특정 조직만을 위한 전용 인프라. 내부 또는 외부 데이터센터에 위치할 수 있음.
- 보안성/통제력: 매우 높음. OS, 하드웨어, 네트워크 구성까지 사용자 정의 가능.
- 비용: 높은 초기 비용 및 유지 관리 비용 필요.
- 예시: 금융사, 병원, 정부 기관 등 고보안 환경 필요 시.

📌 **On-premise와의 차이점**
- 프라이빗 클라우드는 클라우드 원칙(가상화, 온디맨드 자원 확장 등)을 따르며 물리적 위치가 꼭 사내일 필요는 없음.(회사 외부여도 됨)

### 🌍 Public Cloud (퍼블릭 클라우드)
- 특징: AWS, Azure, GCP 등의 공용 클라우드 제공업체 인프라 사용.
- 접근성: 인터넷을 통해 누구나 접근 가능.
- 확장성 & 속도: 즉시 사용 가능, 자동 확장 지원.
- 비용: 초기 투자 없이 사용량 기반 과금 (pay-as-you-go).
- 단점: 데이터센터 위치 등 물리적 제어 불가능.
- 예시 : 스타트업, 예산이 제한된 기업, 빠른 서비스 출시를 원하는 팀.

### 🔀 Hybrid Cloud (하이브리드 클라우드)
- 특징 : 프라이빗 + 퍼블릭 클라우드 조합. 필요에 따라 자원을 분산 운용.
- 대표 시나리오
  - 민감한 데이터(예:환자 정보)는 프라이빗 클라우드에,
  - 분석 및 대시보드는 퍼블릭 클라우드에 처리.

✅ **Cloud Bursting**
- 트래픽 급증 시 프라이빗 클라우드가 감당하지 못하면 **퍼블릭 클라우드로 자동 확장** → Black Friday 같은 이벤트에 유용

### ☁️ 기타 배포 모델

| 모델                  | 설명                           | 예시                   |
| ------------------- | ---------------------------- | -------------------- |
| **Multicloud**      | 여러 클라우드 제공업체를 병행 사용          | AWS로 웹 호스팅 + GCP로 분석 |
| **Community Cloud** | 공통 목적을 가진 조직이 공동으로 사용하는 클라우드 | 연구기관, 정부 부처 등        |

## 2️⃣ 클라우드 인프라와 법적 규제
⚖️ 왜 법이 중요한가요?
클라우드는 전 세계에 분산된 서버를 활용합니다.
이 말은 곧 개인정보가 국경을 넘나들 수 있다는 것이며, 각 국가마다 데이터 보호 법률이 다르기 때문에 기업은 이에 반드시 준수해야 합니다.

### 🌍 사례: 소셜미디어 기업의 글로벌 확장
- 미국 기반 소셜미디어 회사가 아시아, 유럽, 남미에 데이터 센터를 확장
- 호주 사용자가 가입하면, 그 정보는 아시아 서버를 통해 저장될 수 있음
- 그런데!
  - 호주에는 데이터 국외 이전에 관한 법이 있음
  - 아시아 서버가 있는 국가에도 자체 규제가 있음
  - 미국 본사에도 법적 의무가 있음


### 🛡️ 대표적인 규제: GDPR (EU 일반 개인정보 보호법) - General Data Protection Regulation
- 적용 범위: EU 국민의 데이터를 다루는 모든 기업
- 핵심 요구사항
  - 명시적 동의 필요
  - 데이터 유출 시 신속 알림
  - 데이터는 암호화/익명화 필수
  - EU 밖으로 데이터 이전 시 동등한 보호 수준 보장
- 벌금 : 최대 2천만 유로 또는 전세계 연매출의 4%

### 🧾 Personal Data (개인 정보)의 정의
> 개인 정보란 특정 개인을 식별할 수 있는 모든 정보. 개인데이터는 종종 PII(Personally identifiable information) 이라고 불림.

| 예시                   |                   |
| -------------------- | ----------------- |
| 이름, 이메일, 집 주소        | ✅ 식별 가능 정보        |
| IP 주소, 위치 데이터, 건강 정보 | ✅ 간접적 식별 가능 정보    |
| 성적 성향, 인종, 정치 성향 등   | ✅ 민감 정보로 별도 보호 필요 |

| 2위 | Microsoft Azure | **24%** | 기업 친화적, Microsoft 생태계와 통합 강점 |
| 3위 | Google Cloud    | **11%** | 데이터 분석/AI 역량 우수, 멀티클라우드 전략   |

※ 참고: 점유율은 IaaS, PaaS, Hosted Private Cloud 기반

## 2️⃣ AWS (Amazon Web Services)
- 출시: 2006년 (가장 빠름)
- 서비스 수: 175개 이상
- 강점: 확장성, 글로벌 네트워크, 데이터/AI 서비스

### 주요 서비스

| 영역      | 서비스       | 설명                              |
| ------- | --------- | ------------------------------- |
| 저장소     | S3(Simple Storage Service) | 객체 스토리지                         |
| 컴퓨팅     | EC2       | 가상 서버 인스턴스                      |
| 데이터베이스  | RDS       | 관계형 DB (MySQL, PostgreSQL 등 지원) |
| 분석      | Redshift  | 데이터 웨어하우스                       |
| 실시간 데이터 | Kinesis   | 스트리밍 처리                         |
| 머신러닝    | SageMaker | ML 모델 학습/배포 플랫폼                 |


✅ 고객 사례: NerdWallet

- 문제: 모델 배포 지연, 비용 증가
- 해결: SageMaker로 빠른 배포 및 개발비용 75% 절감
- 효과: 모델 개발 기간 수개월 → 수일로 단축

## 3️⃣ Microsoft Azure
- 출시: 2010년
- 강점: 기존 Microsoft 제품과의 통합 (Office, SQL Server 등), 정부/기관 중심 고객층

### 주요 서비스

| 영역     | 서비스              | 설명              |
| ------ | ---------------- | --------------- |
| 저장소    | Blob Storage     | 비정형 데이터 저장      |
| 컴퓨팅    | Virtual Machines | 가상 머신 인프라       |
| 데이터베이스 | Azure SQL        | 클라우드 관계형 DB     |
| 분석     | Microsoft Fabric | 통합 데이터 분석 및 BI  |
| 스트리밍   | Stream Analytics | 실시간 분석          |
| 머신러닝   | Azure ML         | ML 모델 학습/배포 플랫폼 |



✅ 고객 사례: 오타와 병원 (캐나다)
- 문제: 재난 복구 환경 구축 비용 부담
- 해결: Azure Site Recovery 도입 + 클라우드 백업
- 효과:
   - 700TB 백업 지원, 5,300명 동시 사용자 대응
   - 재난복구 비용 약 50% 절감
   - 3개월 만에 구축 완료
   - 캐나다 개인정보보호법(PHIPA) 준수
 
🔗 [Azure 사례 모음](https://www.microsoft.com/ko-kr/customers)

## Google Cloud
- 출시: 2008년
- 강점: 데이터 분석, AI, 멀티 클라우드 중심 전략

### 주요 서비스

| 영역     | 서비스            | 설명                        |
| ------ | -------------- | ------------------------- |
| 저장소    | Cloud Storage  | 객체 저장                     |
| 컴퓨팅    | Compute Engine | 가상 머신                     |
| 데이터베이스 | Cloud SQL      | RDB 서비스                   |
| 분석     | BigQuery       | 초대용량 분석 플랫폼               |
| 스트리밍   | Dataflow       | 스트리밍 분석 처리                |
| AutoML | 머신러닝 자동화 도구    | 비전, 자연어 처리 등 자동화 모델 생성 가능 |


✅ 고객 사례: Lush (영국 화장품 브랜드)
- 문제: Boxing Day(영국 최대 세일) 트래픽 폭주 → 서비스 중단
- 해결: Google Cloud로 17개 사이트 및 모바일 앱 전체 마이그레이션
- 효과:
   - 트래픽 폭주 대응 성공
   - 호스팅 비용 40% 절감
   - 이미지 인식 앱 배포로 고객 경험 향상

🔗 [Lush 사례 링크](https://cloud.google.com/customers/lush?hl=ko)
